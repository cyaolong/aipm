# 00 Brief

## 已确认信息
- 产品定位：面向产品经理的自动化助手，用户输入 idea 后，自动完善 idea、输出 PRD，并生成 iOS 26 风格的低保真原型。
- 平台：MVP 为 Web 端，验证后再扩展 App/小程序。
- 商业目标：营收优先，需有付费用户才算验证通过；可提供免费额度引流。
- 指标目标：DAU 50；月收入 1000 元 RMB。
- 目标用户与场景：产品经理在日常工作中需要更快完成构想完善与文档/原型产出，以提升效率、减轻压力。
- 约束：1-2 天内完成开发；单人执行；无开发能力。

## 初步假设
- 用户愿意以“文档 + 原型”一体化产出为价值点付费。
- MVP 以“流程清晰、产出结构完整”为核心体验，暂不追求高度定制。
- 以 Web 端为起点即可覆盖多数 PM 使用场景。

## 待确认问题
- 目标用户的付费意愿价格区间（按次 / 订阅 / 包月）
- 免费额度策略（额度口径、限制方式）
- 期望的 PRD 深度（轻量 / 标准 / 深度）
- 是否需要账号体系与历史项目管理（MVP 是否要做）
- 原型输出是否需要下载导出（图片/PDF）还是仅 Figma 链接
